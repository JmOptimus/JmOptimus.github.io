Es el comando análogo a grep en linux.

Devuelve solo aquellas líneas que contengan una cadena de caracteres determinada.
Normalmente se usa conjuntamente con tuberías y otro comando que le proporcione el texto original a filtrar. (Ver la sección de Tuberías)

Modo de uso:
comando-1 | findstr cadena-de-caracteres

dir | findstr log

El comando mostrado arriba buscará dentro del resultado del comando "dir" (obtener el listado de ficheros/directorios del directorio actual), solo aquellas lineas que contengan la palabra "log" y las mostrará por pantalla.

Estamos acostumbrados a escribir un comando en la consola y que,
por defecto, nos muestre el resultado por pantalla.

Con los separadores que se muestran a continuación seremos capaces de
redirigir el cauce natural de esa salida hacia donde más nos convenga.

Si queremos que todo lo que saldría normalmente por pantalla tras ejecutar el comando, sea escrito a un fichero destino,

Usaremos el separador > de la siguiente manera:

comando > ruta-fichero-destino

dir > contenido-directorio.txt

ls > contenido-directorio.txt

Con el comando de arriba se almacenará, en "contenido-directorio.txt", el listado de ficheros y directorios contenidos en el directorio actual."

Imaginemos que queremos ir registrando en un fichero de log una serie de eventos.
En este caso nos interesa ir concatenando cada nuevo registro en un historial sin sobreescribir la información ya existente.

Para este caso particular usaremos el separador >> de la siguiente manera:

comando >> ruta-fichero-destino

date >> log.txt

Cada vez que se ejecute el comando de arriba, se añadirá al final del fichero log.txt la fecha y hora de ese mismo instante."

Es uno de los separadores más potentes ya que permite comunicar dos comandos/programas y puede encadenarse indefinidamente.

La tubería conecta la salida del primer comando con la entrada del segundo.

Para construir una tubería entre dos comandos sólo tendremos que escribir | en el medio de ambos:

comando-1 | comando-2

cat security-log.txt | grep warning

El comando de arriba buscaría dentro de un fichero llamado security-log.txt aquellas lineas que contuvieran la palabra "warning".
Finalmente lo mostraría por pantalla.

Sirve para saber si un host es accesible desde nuestro host;
siempre y cuando el host que consultemos no tenga bloqueado el protocolo ICMP.

Modo de uso:

ping ip-host-destino
ping nombre-host-destino

ping 10.1.4.11

Comprobaría si el host 10.1.4.11 es alcanzable desde mi host.

ping 10.1.4.11 -t

Al añadir el parámetro -t, la ejecución del comando no parará hasta que el usuario lo decida.

Es de gran utilidad ya que sirve para monitorizar los puertos que están abiertos en un momento dado, junto con la dirección IP del host que está escuchando en ellos.

netstat -an | findstr ":80"

netstat -an | grep ":80"

Comprobaría qué hosts están escuchando en el puerto 80.

netstat -o

Al añadir el parámetro -o, también mostraría el número de identificación del proceso (PID); para poder matarlo después (Ver comando kill/taskkill).

Sirve para enviar de forma segura (SSH) ficheros/directorios remotamente.
Para ello es necesario que el servidor SSH esté activo (Ver activación SSH).

Si no se especifica un puerto, por defecto se usará el 22.
Será necesario autenticarse con el usuario especificado.

Modo de uso:
scp ruta-fichero nombre-usuario@IP-destino:ruta-destino scp -P puerto ruta-fichero nombre-usuario@IP-destino:ruta-destino

scp virus.exe juan@10.1.4.10:c:\Windows\System32\

Enviaría el archivo virus.exe al directorio c:\Windows\System32\ del host ubicado en 10.1.4.10 con usuario "juan"

scp -r /peliculas/ almodovar@10.1.4.10:c:\Users\Documents\Peliculas\

Enviaría el directorio (con todo su contenido) a la ruta c:\Users\Documents\Peliculas\ del host ubicado en 10.1.4.10 con usuario "almodovar".

Sirve para abrir una consola remota de forma segura (SSH).
Para ello es necesario que el servidor SSH esté activo (Ver activación SSH).

Si no se especifica nada, el puerto por defecto es el 22.
Será necesario autenticarse con el usuario especificado.

Modo de uso:
ssh nombre-usuario@IP-destino ssh -p puerto nombre-usuario@IP-destino

ssh tarde@10.1.4.10

Se conectaría remotamente al dispositivo ubicado en 10.1.4.10 autenticándose como el usuario "tarde".
